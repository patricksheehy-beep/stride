    2. src/scoring/factors/trail-preference.js:
       - Export HIGHWAY_PREFERENCE object mapping OSM highway tag values to 0-1 scores. Use exact values from research: path=1.0, footway=0.90, track=0.85, bridleway=0.80, cycleway=0.70, pedestrian=0.65, steps=0.40, living_street=0.35, residential=0.20, unclassified=0.15, service=0.10, tertiary=0.05.        - Export scoreTrailPreference(routeGeoJSON, trailData) that iterates trailData.features, reads feature.properties.highway, looks up score in HIGHWAY_PREFERENCE (default 0.3), computes length-weighted average using @turf/length. Returns number 0-1.

    3. src/scoring/factors/continuity.js:
       - Export scoreContinuity(routeGeoJSON) that takes the route GeoJSON (FeatureCollection with one LineString feature). Extract coordinates from the first feature's geometry. For each consecutive triplet of coordinates, compute bearing change using @turf/bearing. Count turns where absolute bearing change exceeds 120 degrees as "sharp turns". Score = 1.0 - (sharpTurns / totalCoordinates), clamped to [0, 1]. Handle edge cases: less than 3 coordinates returns 1.0.

    4. src/scoring/factors/scenic.js:
       - Export scoreScenic(routeGeoJSON, trailData) that computes a scenic value from available OSM data. Three sub-signals:
         (a) Water proximity: count features in trailData where properties contain natural=water, waterway=stream/river/canal, or name contains "river/creek/lake/pond". Score = min(waterFeatureCount / 3, 1.0) * 0.4.
         (b) Green proximity: count features where properties contain leisure=park/garden/nature_reserve, landuse=forest/grass/meadow, or natural=wood/grassland. Score = min(greenFeatureCount / 5, 1.0) * 0.3.
         (c) Named trail bonus: count features with routeType property (relation members from Overpass). Score = min(namedTrailCount / 4, 1.0) * 0.3.
         Total scenic = water + green + namedTrail. Base score when nothing found = 0.3. Clamp to [0, 1].

    Tests use mock GeoJSON data -- construct FeatureCollection with LineString features having the relevant properties. Do NOT call external APIs. Use @turf/helpers for GeoJSON construction.
