    1. EXTEND src/routing/adapters/ors.js -- add roundTrip method to existing ORSAdapter class:
       ```javascript
       async roundTrip(startPoint, options = {}) {
         const { length = 5000, points = 5, seed = 0 } = options;
         const body = {
           coordinates: [[startPoint.lng, startPoint.lat]],
           preference: 'recommended',
           units: 'km',
           geometry: true,
           instructions: false,
           options: {
             round_trip: {
               length: length,   // Already in meters from caller
               points: points,
               seed: seed
             },
             profile_params: {
               weightings: {
                 green: { factor: 1.0 },
                 quiet: { factor: 1.0 }
               }
             }
           }
         };
         const response = await fetch(
           `${this.baseUrl}/v2/directions/foot-hiking/geojson`,
           {
             method: 'POST',
             headers: {
               'Authorization': this.apiKey,
               'Content-Type': 'application/json'
             },
             body: JSON.stringify(body)
           }
         );
         if (response.status === 429) throw new Error('ORS rate limit exceeded');
         if (!response.ok) throw new Error(`ORS round trip error: ${response.status}`);
         return response.json();
       }
       ```
       IMPORTANT: Keep existing buildRequestBody and route methods unchanged. The roundTrip method is ADDITIVE.

    2. CREATE src/routing/route-builder.js:
       Import: @turf/length, @turf/bearing, @turf/destination, @turf/nearest-point-on-line, @turf/helpers (point, featureCollection), eventBus from core.

       Export class RouteBuilder:
       - constructor({ orsAdapter, engineManager, scorer }) stores all three

       - async generateCandidatesViaRoundTrip(startPoint, distanceKm, count = 3):
         Loop seed from 0 to count+1 (generate count+2 attempts). For each seed, call this.orsAdapter.roundTrip(startPoint, { length: distanceKm * 1000, points: 5, seed }). Catch errors per-seed (some seeds fail). Collect successful results. Return first `count` results. If fewer than count succeed, return what we have.

       - buildLoopWaypoints(startPoint, trails, targetDistanceKm):
         Calculate radius = targetDistanceKm / (2 * Math.PI). Place 4 waypoints at bearings [0, 90, 180, 270] degrees from startPoint at `radius` km using @turf/destination. For each waypoint, snap to nearest trail using @turf/nearest-point-on-line against trail features. Convert snapped points back to { lat, lng } format. Return array of 4 waypoints.

       - async generateCandidateViaWaypoints(startPoint, trailData, distanceKm):
         Call buildLoopWaypoints to get 4 snapped waypoints. Build full waypoint array: [startPoint, wp1, wp2, wp3, wp4, startPoint] (loop back to start). Call this.engineManager.route(fullWaypoints). Return the result.

       - async refineDistance(startPoint, targetKm, maxIterations = 3):
         Let adjustedTarget = targetKm. Loop up to maxIterations times:
           Generate 1 candidate via roundTrip with adjustedTarget.
           Measure actual distance using @turf/length on the first feature's geometry (units: 'kilometers').
           Compute error = abs(actualKm - targetKm) / targetKm.
           If error <= 0.10, return the candidate (within tolerance).
           Otherwise, compute ratio = targetKm / actualKm, set adjustedTarget *= ratio, continue.
         After loop exhausts, return best candidate found (closest to target).

       Emit events via eventBus:
       - 'route:candidates-generating' when starting generation
       - 'route:candidate-generated' for each successful candidate with { index, distanceKm }
