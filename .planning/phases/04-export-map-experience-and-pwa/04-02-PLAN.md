  <action>
    1. Create src/scoring/scoring-worker.js:
       - ESM module worker (Vite handles bundling with `{ type: 'module' }`)
       - Import RouteScorer from './scorer.js' (relative import -- Vite resolves in worker context)
       - `self.onmessage = function(e)`:
         - Destructure: `{ candidates, startPoint, weights }` from e.data
         - Create RouteScorer with weights
         - Call scorer.scoreAndRank(candidates, startPoint)
         - postMessage the ranked results array
       - Wrap in try/catch; on error, postMessage `{ error: e.message }`

    2. Modify src/routing/route-generator.js:
       - Add a private method `_initWorker()` called once in constructor:
         - Try to create worker: `this._worker = new Worker(new URL('../scoring/scoring-worker.js', import.meta.url), { type: 'module' })`
         - Catch: set `this._worker = null` (graceful degradation for environments without Worker support)
       - Add a private method `_scoreInWorker(candidates, startPoint, weights)`:
         - Returns a Promise
         - Sets worker.onmessage to resolve with e.data (check for e.data.error and reject if present)
         - Sets worker.onerror to reject
         - Calls worker.postMessage({ candidates, startPoint, weights })
         - Add a 30-second timeout that rejects (prevents hanging if worker crashes)
       - In generate() method, replace the direct `scorer.scoreAndRank(candidates, startPoint)` call:
         - If this._worker exists, try `await this._scoreInWorker(candidates, startPoint, finalWeights)`
         - Catch: fall back to `scorer.scoreAndRank(candidates, startPoint)` on main thread
         - If this._worker is null, use main thread scorer directly (existing behavior)
       - Constructor: add `_initWorker()` call. Keep existing scorer parameter for fallback.

         - Score with RouteScorer.scoreAndRank on main thread
         - Verify the scoring logic is deterministic (same input = same output)
         - Verify sorting is descending by total
       - Add a test that RouteGenerator constructor does not throw when Worker is unavailable (mock Worker to throw)
