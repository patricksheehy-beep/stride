    Export class `RouteExplainer`:
    - Constructor takes `claudeClient` (a ClaudeClient instance)
    - Method `_buildExplainPrompt(routeData)`:
      - Takes `{ distanceKm, score: { total, breakdown }, trailNames, surfaces, waterFeatures, greenSpaces, vibeKeywords }` (all optional except score)
      - Builds a structured prompt string describing the route:
        ```
        Route: {distanceKm}km loop
        Overall score: {total}/1.0

        Score breakdown:
        - Surface quality: {surfaceScore}/1.0
        - Route continuity: {continuityScore}/1.0
        - Trail preference: {trailPrefScore}/1.0
        - Scenic value: {scenicScore}/1.0
        - Green space: {greenSpaceScore}/1.0

        Trail names: {trailNames.join(', ') || 'unnamed trails'}
        Surfaces: {surfaces.join(', ') || 'mixed'}
        Water features: {waterFeatures.join(', ') || 'none nearby'}
        Green spaces: {greenSpaces.join(', ') || 'none nearby'}
        User vibe: {vibeKeywords.join(', ') || 'general run'}
        ```
      - Return the prompt string

    - Method `async explain(routeData)`:
      - Build prompt via _buildExplainPrompt(routeData)
      - Wrap in try/catch
      - Call `this.claudeClient.complete(EXPLAINER_SYSTEM_PROMPT, prompt, null, { maxTokens: 256 })`
      - Return the text response (no structured output needed -- free text)
      - On error: return fallback string: `"A ${routeData.distanceKm || ''}km route scoring ${routeData.score?.total?.toFixed(2) || 'N/A'} overall."` (never return null)

    - Method `async explainBatch(routeDataArray, maxRoutes = 3)`:
      - Take the first maxRoutes items from routeDataArray
      - Build a combined prompt with all routes numbered:
        ```
        Generate a brief 2-3 sentence explanation for each of the following routes:

        Route 1:
        {_buildExplainPrompt(routeDataArray[0])}

        Route 2:
        {_buildExplainPrompt(routeDataArray[1])}
        ...
        ```
      - System prompt: "For each route, write a separate explanation paragraph. Label each 'Route 1:', 'Route 2:', etc."
      - Call Claude once with the combined prompt (no structured output, maxTokens: 512)
      - Parse response by splitting on "Route N:" patterns
      - Return array of explanation strings
      - On error: return array of fallback explanations for each route

    - Method `_extractTrailMetadata(routeGeoJSON, trailData, landUseData)`:
      - Helper that extracts trail names, surface types, water features, and green space names from GeoJSON properties
      - Trail names: unique `name` or `relationName` values from trailData features
      - Surfaces: unique `surface` values from trailData features
      - Water features: features where `natural === 'water'` or `waterway` is set -- extract names
      - Green spaces: features from landUseData where `leisure` or `landuse` is set -- extract names
      - Returns `{ trailNames, surfaces, waterFeatures, greenSpaces }`
