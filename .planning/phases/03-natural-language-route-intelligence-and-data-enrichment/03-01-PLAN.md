<objective>
Build the land-use data pipeline and green space scoring factor that enables Stride to evaluate routes based on how much of their length passes through parks, forests, water areas, and nature reserves.

Purpose: DATA-03 (multi-source data fusion) and DATA-04 (green space scoring) are the data foundation that both NL parsing and route explanations depend on. The current scenic.js only counts nearby features -- it does not calculate what percentage of a route actually passes through green space. This plan adds true geometric intersection scoring.

Output: Overpass land-use query, polygon normalization, green space scoring factor, and 5-factor weight expansion. All tested.
</objective>

    **Step 1: Create src/scoring/factors/green-space.js**
    Create the green space scoring factor following the pattern from scenic.js:

    ```javascript
    import along from '@turf/along';
    import { booleanPointInPolygon } from '@turf/boolean-point-in-polygon';
    import length from '@turf/length';
    ```

    Export function `scoreGreenSpace(routeGeoJSON, landUseFeatures)`:
    - If routeGeoJSON has no features OR landUseFeatures is null/undefined/empty, return 0.4 (neutral base, per pitfall 4 in research)
    - Extract the first feature from routeGeoJSON (the route LineString)
    - Calculate route length in km using @turf/length
    - If route length is 0, return 0.4
    - Sample points every 0.1 km (100m) along the route using @turf/along
    - Filter landUseFeatures to only Polygon and MultiPolygon geometry types
    - For each sample point, check if it falls inside ANY green polygon using @turf/boolean-point-in-polygon
    - Count greenSamples (points inside at least one polygon)
    - Calculate greenRatio = greenSamples / totalSamples
    - Return score: 0.2 + greenRatio * 0.8 (scale: 0% green = 0.2, 100% green = 1.0)
